title: react-overlay-stack
description: Type-safe, unstyled overlay manager for React 19 with animation lifecycle and stacking support.
---

A type-safe, unstyled overlay manager for React 19. Built on [Zustand](https://github.com/pmndrs/zustand) for state management.

## Features

- **Type-safe** — Full TypeScript support with inferred props and result types via phantom types
- **Unstyled** — Bring your own UI components (Radix, Vaul, custom, etc.)
- **Stacking** — Multiple overlays stack naturally with metadata (`stackIndex`, `isTopmost`)
- **Animation lifecycle** — Built-in `isOpen` and `onAnimationEnd` for enter/exit transitions
- **Independent systems** — Create isolated overlay systems with `createOverlaySystem()`

## Quick Example

```tsx
import { createOverlaySystem, defineOverlay } from "react-overlay-stack";

const { useOverlay, OverlayProvider } = createOverlaySystem();

const ConfirmDialog = defineOverlay<{ message: string }, boolean>(
  ({ message, controller }) => (
    <dialog open>
      <p>{message}</p>
      <button onClick={() => controller.close(true)}>OK</button>
      <button onClick={() => controller.close(false)}>Cancel</button>
    </dialog>
  ),
);

const App = () => {
  const confirm = useOverlay(ConfirmDialog);

  return (
    <OverlayProvider>
      <button onClick={() => confirm.open({ message: "Are you sure?" })}>
        Open
      </button>
    </OverlayProvider>
  );
};
```
````

## Setup

### 1. Create an overlay system

Create a shared file that exports your overlay system:

```ts
// overlay-system.ts
import { createOverlaySystem } from "react-overlay-stack";

export const { useOverlay, OverlayProvider } = createOverlaySystem();
```

Each call to `createOverlaySystem()` creates an independent overlay stack with its own state. Most apps need one.

### 2. Add the provider

Wrap your app (or a subtree) with `OverlayProvider`:

```tsx
import { OverlayProvider } from "./overlay-system";

const App = () => (
  <OverlayProvider>
    <YourApp />
  </OverlayProvider>
);
```

### 3. Define an overlay

Use `defineOverlay()` to create a typed overlay component. The first generic is the props type, the second is the result type:

```tsx
import { defineOverlay } from "react-overlay-stack";

const AlertDialog = defineOverlay<{ message: string }>(
  ({ message, controller }) => (
    <dialog open>
      <p>{message}</p>
      <button onClick={() => controller.close()}>OK</button>
    </dialog>
  ),
);
```

### 4. Open it

Use the `useOverlay` hook to get an `open` function:

```tsx
import { useOverlay } from "./overlay-system";

const Page = () => {
  const alert = useOverlay(AlertDialog);

  return (
    <button onClick={() => alert.open({ message: "Hello!" })}>
      Show Alert
    </button>
  );
};
```

### 5. Get results

Pass a callback as the second argument to `open`, or use a typed result:

```tsx
const ConfirmDialog = defineOverlay<{ message: string }, boolean>(
  ({ message, controller }) => (
    <dialog open>
      <p>{message}</p>
      <button onClick={() => controller.close(true)}>Yes</button>
      <button onClick={() => controller.close(false)}>No</button>
    </dialog>
  ),
);

// In your component:
const confirm = useOverlay(ConfirmDialog);
confirm.open({ message: "Delete item?" }, (result) => {
  if (result) {
    deleteItem();
  }
});
```

## `createOverlaySystem()`

Factory function that creates an independent overlay system.

```ts
import { createOverlaySystem } from "react-overlay-stack";

const { useOverlay, OverlayProvider } = createOverlaySystem();
```
````

**Returns:**

| Property          | Type                                                          | Description                                       |
| ----------------- | ------------------------------------------------------------- | ------------------------------------------------- |
| `useOverlay`      | `(overlay: OverlayDefinition) => UseOverlayReturn`            | Hook to control an overlay                        |
| `OverlayProvider` | `FC<{ children: ReactNode; overlays?: OverlayDefinition[] }>` | Provider component that renders the overlay stack |

### `OverlayProvider`

Wrap your component tree with `OverlayProvider` to render overlays. It accepts:

| Prop       | Type                  | Description                                                                  |
| ---------- | --------------------- | ---------------------------------------------------------------------------- |
| `children` | `ReactNode`           | Your application content                                                     |
| `overlays` | `OverlayDefinition[]` | Optional array of overlays to register eagerly (instead of via `useOverlay`) |

### `useOverlay(overlay)`

Hook that registers an overlay and returns controls.

**Parameters:**

| Parameter | Type                                 | Description                                         |
| --------- | ------------------------------------ | --------------------------------------------------- |
| `overlay` | `OverlayDefinition<TProps, TResult>` | The overlay definition created by `defineOverlay()` |

**Returns: `UseOverlayReturn<T>`**

| Method     | Signature                                                         | Description                                                                                       |
| ---------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| `open`     | `(props: TProps, onClose?: (result?: TResult) => void) => string` | Open the overlay. Returns the instance ID. If `TProps` is `never`, the first argument is omitted. |
| `close`    | `(id: string, result?: TResult) => void`                          | Close a specific overlay instance by ID                                                           |
| `closeAll` | `() => void`                                                      | Close all open overlays in this system                                                            |

---

## `defineOverlay()`

Creates a typed overlay definition.

```ts
import { defineOverlay } from "react-overlay-stack";

const MyOverlay = defineOverlay<TProps, TResult>(component);
```

**Type Parameters:**

| Parameter | Default     | Description                                            |
| --------- | ----------- | ------------------------------------------------------ |
| `TProps`  | `never`     | Props passed to the overlay (excluding `controller`)   |
| `TResult` | `undefined` | Type of the value passed to `controller.close(result)` |

**Parameters:**

| Parameter   | Type                                | Description                   |
| ----------- | ----------------------------------- | ----------------------------- |
| `component` | `OverlayComponent<TProps, TResult>` | The React component to render |

**Returns:** `OverlayDefinition<TProps, TResult>`

The component receives all `TProps` plus a `controller` prop:

```ts
interface OverlayController<TResult> {
  readonly close: (result?: TResult) => void;
}
```

---

## `useOverlayContext()`

Hook to access overlay metadata from within an overlay component.

```ts
import { useOverlayContext } from "react-overlay-stack";

const MyOverlay = defineOverlay(() => {
  const { isOpen, stackIndex, isTopmost, onAnimationEnd } = useOverlayContext();
  // ...
});
```

**Returns: `OverlayMetadata`**

| Property         | Type         | Description                                                                |
| ---------------- | ------------ | -------------------------------------------------------------------------- |
| `id`             | `string`     | Unique instance ID                                                         |
| `stackIndex`     | `number`     | Zero-based position in the overlay stack                                   |
| `isTopmost`      | `boolean`    | Whether this overlay is the topmost in the stack                           |
| `isOpen`         | `boolean`    | `true` when visible, `false` when closing (use for exit animations)        |
| `onAnimationEnd` | `() => void` | Call this when exit animation completes to remove the overlay from the DOM |

---

## Types

### `OverlayDefinition<TProps, TResult>`

Opaque type returned by `defineOverlay()`. Carries phantom types for compile-time safety.

### `OverlayProps<T>`

Utility type to extract props from an overlay definition:

```ts
type Props = OverlayProps<typeof MyOverlay>; // { message: string }
```

### `OverlayResult<T>`

Utility type to extract the result type from an overlay definition:

```ts
type Result = OverlayResult<typeof ConfirmDialog>; // boolean
```

### `OverlayComponent<TProps, TResult>`

The component type that overlays must satisfy. When `TProps` is `never`, only `controller` is received. Otherwise, the component receives `TProps & { controller: OverlayController<TResult> }`.

### `OverlayController<TResult>`

```ts
interface OverlayController<TResult> {
  readonly close: (result?: TResult) => void;
}
```

### `OverlayMetadata`

```ts
interface OverlayMetadata {
  readonly id: string;
  readonly stackIndex: number;
  readonly isTopmost: boolean;
  readonly isOpen: boolean;
  readonly onAnimationEnd: () => void;
}
```

const BasicPanel = defineOverlay<{ title: string }, string>(

title: Radix UI Dialog
description: How to integrate react-overlay-stack with Radix UI Dialog.
---

[Radix UI Dialog](https://www.radix-ui.com/primitives/docs/components/dialog) is a popular accessible dialog primitive. Here's how to wrap it as an overlay.

## Creating a Modal Component

Build a compound component that reads overlay metadata via `useOverlayContext()`:

```tsx
import * as Dialog from "@radix-ui/react-dialog";
import { useOverlayContext } from "react-overlay-stack";
import type { FC, ReactNode } from "react";

interface ModalProps {
  children: ReactNode;
}

export const Modal: FC<ModalProps> = ({ children }) => {
  const { isOpen, stackIndex, onAnimationEnd } = useOverlayContext();

  const overlayZIndex = 50 + stackIndex * 10;
  const contentZIndex = overlayZIndex + 1;

  return (
    <Dialog.Root open={true}>
      <Dialog.Portal>
        <Dialog.Overlay
          className="fixed inset-0 bg-black/50"
          data-state={isOpen ? "open" : "closed"}
          style={{ zIndex: overlayZIndex }}
        />
        <Dialog.Content
          className="fixed left-1/2 top-1/2 w-full max-w-md -translate-x-1/2 -translate-y-1/2 rounded-lg bg-white p-6 shadow-xl"
          data-state={isOpen ? "open" : "closed"}
          style={{ zIndex: contentZIndex }}
          onAnimationEnd={(event) => {
            if (event.currentTarget === event.target && !isOpen) {
              onAnimationEnd();
            }
          }}
        >
          {children}
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  );
};
```
````

**Key points:**

- `Dialog.Root` always has `open={true}` — the overlay system controls visibility, not Radix.
- `data-state` is driven by `isOpen` from context for CSS animations.
- `onAnimationEnd` fires after exit animation to remove the overlay from the DOM.
- Z-index is computed from `stackIndex` so overlays stack correctly.

## Defining the Overlay

```tsx
import { defineOverlay } from "react-overlay-stack";
import { Modal } from "./Modal";

const ConfirmDialog = defineOverlay<{ message: string }, boolean>(
  ({ message, controller }) => (
    <Modal>
      <h2 className="text-lg font-semibold">Confirm</h2>
      <p className="my-4 text-gray-600">{message}</p>
      <div className="flex gap-2">
        <button onClick={() => controller.close(true)}>Yes</button>
        <button onClick={() => controller.close(false)}>No</button>
      </div>
    </Modal>
  ),
);
```

## Usage

```tsx
const Page = () => {
  const confirm = useOverlay(ConfirmDialog);

  const handleDelete = () => {
    confirm.open({ message: "Delete this item?" }, (result) => {
      if (result) {
        deleteItem();
      }
    });
  };

  return <button onClick={handleDelete}>Delete</button>;
};
```

```mdx
---
title: Overlay Stacking
description: How overlay stacking works in react-overlay-stack.
---

When multiple overlays are open at the same time, they form a stack. Each overlay knows its position and can adapt its behavior accordingly.

## How It Works

Every overlay rendered by `OverlayProvider` receives metadata via `useOverlayContext()`:

| Property | Description |
|----------|-------------|
| `stackIndex` | Zero-based position. First overlay is `0`, second is `1`, etc. |
| `isTopmost` | `true` only for the overlay at the top of the stack |
| `isOpen` | `true` when visible, `false` when animating out |

## Z-Index Management

Use `stackIndex` to compute z-index so overlays layer correctly:

```tsx
const overlayZIndex = 50 + stackIndex * 10;
const contentZIndex = overlayZIndex + 1;
````

This gives each overlay a 10-unit z-index gap, enough for overlay + content layers.

## Closing Behavior

The overlay system provides three ways to close overlays:

```tsx
const overlay = useOverlay(MyOverlay);

// Close a specific instance by ID
const id = overlay.open(props);
overlay.close(id);

// Close with a result
overlay.close(id, "confirmed");

// Close all overlays in this system
overlay.closeAll();
```

## Animation Lifecycle

When `close()` is called, the overlay is **not** immediately removed from the DOM. Instead:

1. `isOpen` becomes `false` — trigger your exit animation
2. When the animation finishes, call `onAnimationEnd()`
3. The overlay is removed from the DOM and the `onClose` callback fires

```tsx
const MyOverlay = defineOverlay(({ controller }) => {
  const { isOpen, onAnimationEnd } = useOverlayContext();

  return (
    <div
      className="transition-opacity duration-200"
      style={{ opacity: isOpen ? 1 : 0 }}
      onTransitionEnd={() => {
        if (!isOpen) onAnimationEnd();
      }}
    >
      <button onClick={() => controller.close()}>Close</button>
    </div>
  );
});
```

If you don't need exit animations, call `onAnimationEnd()` immediately:

```tsx
import { useEffect } from "react";

const SimpleOverlay = defineOverlay(({ controller }) => {
  const { isOpen, onAnimationEnd } = useOverlayContext();

  useEffect(() => {
    if (!isOpen) onAnimationEnd();
  }, [isOpen, onAnimationEnd]);

  return <div>...</div>;
});
```

## General Rules

- **Arrow functions everywhere** — no `function` declarations

  ```ts
  // Good
  const add = (a: number, b: number) => a + b;

  // Bad
  function add(a: number, b: number) {
    return a + b;
  }
````

- **Named exports only** — no default exports

  ```ts
  // Good
  export const MyComponent: FC = () => { ... };

  // Bad
  export default function MyComponent() { ... }
  ```

- **Component typing** — use `FC<Props>` with destructured props

  ```tsx
  interface ButtonProps {
    label: string;
    onClick: () => void;
  }

  const Button: FC<ButtonProps> = ({ label, onClick }) => (
    <button onClick={onClick}>{label}</button>
  );
  ```
